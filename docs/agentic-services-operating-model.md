# Agentic Services Operating Model

## Purpose

This document extends DelEx-OS from delivery governance into a governed AI-agent consulting and managed-services operating model. The point is not to treat AI as a single specialist workstream. The point is to define how agentic work is scoped, supervised, contracted, evaluated, and expanded without losing human accountability.

## Core rule

Autonomy is downstream of specification quality, dependency readiness, policy clarity, and evaluation maturity.

Agents should own throughput.
Humans should own ambiguity, authority, and accountability.
Policies should own permissions.
Evidence should own scale decisions.

## Delivery cells

### 1. Joint Steering and Governance Cell
Owns business outcomes, risk posture, commercial scope, release authority for high-consequence capabilities, and escalation authority.

### 2. Delivery Control Office
Owns milestone control, dependency sequencing, access readiness, evaluation gates, release readiness, and incident coordination.

### 3. Process and Decision Engineering
Maps real work into governed decision loops:
- trigger
- required context
- tools and constraints
- expected output
- approval rule
- exception path
- KPI and emitted evidence

### 4. Systems Integration and Agent Orchestration
Owns tool wrappers, workflow engines, API contracts, identity boundaries, retries, failure routing, and execution topology.

### 5. Knowledge, Data, and Retrieval Engineering
Owns corpus shaping, metadata quality, retrieval strategy, schemas, freshness rules, semantic indexing, and evidence packaging.

### 6. Agent Systems Engineering
Owns agent boundaries, routing logic, model selection, prompt/policy composition, memory scope, and coordination behavior.

### 7. Evaluation, Assurance, and Runtime Operations
Owns offline evaluation, runtime telemetry, policy violation detection, intervention-rate tracking, incident handling, rollback, and drift management.

### 8. Experience and Human Intervention Design
Owns review surfaces, approval queues, exception handling, operator dashboards, override controls, and audit views.

### 9. Security, Identity, and Policy
Owns access boundaries, data classification, action permissions, segregation of duties, and policy-as-code direction.

### 10. Change, Adoption, and Capability Transfer
Owns operator adoption, enablement, SOP integration, service handoff, and ongoing value realization.

## Minimum human-accountable roles

Every governed agentic engagement must name:
- sponsor
- business or process owner
- system owner
- data or knowledge owner
- policy or risk owner
- operator representative
- delivery lead
- principal architect
- evaluation lead

If these roles are absent, the project is under-governed even if the build looks technically successful.

## Control-loop model

The unit of design is not a chatbot response. It is a governed control loop:
- trigger
- context
- agent work
- action class
- approval rule
- exception route
- emitted evidence
- KPI outcome
