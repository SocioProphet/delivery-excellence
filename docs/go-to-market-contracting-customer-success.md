# Go-to-Market, Contracting, and Customer Success

## Commercial stance

We should not sell agentic services as vague autonomy or seat count. We should sell a phased governed-service program.

## Service staircase

### Stage 1 — Readiness and Design
Produces:
- workflow and dependency baseline
- action-class schedule
- policy boundary draft
- evaluation and KPI design
- delivery plan

Commercial form:
- fixed-fee diagnostic and architecture package

### Stage 2 — Build and Shadow
Produces:
- one bounded control loop
- review surfaces
- runtime telemetry
- shadow-mode comparison evidence

Commercial form:
- milestone-based implementation package

### Stage 3 — Controlled Action Pilot
Produces:
- approved A2/A3 subset
- rollback and incident runbooks
- exception operations
- pilot KPI and assurance review

Commercial form:
- milestone package or short pilot retainer

### Stage 4 — Managed Assurance and Expansion
Produces:
- regression governance
- drift review
- policy updates
- autonomy expansion proposals
- quarterly value review

Commercial form:
- managed services retainer

## Contracting implications

Contracts should explicitly define:
- action classes
- approval rules
- prohibited actions
- client-side named owners
- access and corpus dependencies
- evaluation and acceptance schedule
- rollback rights
- logging and audit expectations
- change-control rules for widening autonomy

## Customer success operating rhythm

### Weekly operating review
- intervention rate
- unresolved exceptions
- dependency outages
- tuning backlog

### Monthly assurance review
- regression results
- drift indicators
- policy violations
- knowledge freshness
- operator trust signals

### Quarterly value review
- KPI movement
- realized value
- scope expansion candidates
- autonomy-envelope revision proposal
- renewal or managed-service path

## Why this belongs in DelEx-OS

The shift from ordinary delivery to governed agentic services changes how we scope work, define ownership, contract responsibilities, and measure customer success. DelEx-OS should therefore carry both delivery governance and the method by which delivery turns into repeatable services.
