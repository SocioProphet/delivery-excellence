# DelEx-OS Scope and Purpose

DelEx-OS exists to make delivery execution across the platform and its ecosystem inspectable, decidable, auditable, and repeatable. We operationalize “excellence” as a system property created by constraints (scope gates), feedback loops (KPIs), and durable evidence (templates + ADRs + artifacts).

Scope is enforced via inclusion tests (capacity / risk / cross-team dependency) and explicit exclusions (FYI-only chatter, tiny local tasks with no reuse value, speculative ideas without a DoD). Governance is GitOps-first: changes to standards, routing, automation, or architecture require reviewable commits plus recorded rationale.

This repository is the “governance brain” (definitions, templates, scoreboards). Automation bindings (groups, email, calendars, routing validation) live in the companion repo: SocioProphet/delivery-excellence-automation.
