# ADRs

ADRs (Architecture Decision Records) capture decisions that change:
- architecture, interfaces, data contracts
- security posture or control gates
- standards/templates/routing/automation
- anything with cross-team or cross-GEO impact

Rule: if we can argue about it later, we must record it now.

Use adr/template.md and place ADRs under adr/decisions/NNNN-title.md.
